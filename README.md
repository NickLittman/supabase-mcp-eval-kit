# Supabase MCP Eval Sandbox

A pre-populated Supabase project for testing MCP tool calls across the full surface area: SQL execution, RLS, storage, Edge Functions, pgvector, triggers, views, and schema introspection.

## Quick Start

### One command (recommended)

```bash
supabase login     # if not already logged in
./setup.sh --org-id <your-org-id>
```

This creates the project, applies the migration, and deploys the Edge Function. Run `./setup.sh --help` for options.

### Manual setup

```bash
# 1. Create project
supabase projects create mcp-eval-sandbox \
  --org-id <org-id> --region us-east-1 --db-password <pw>

# 2. Link + push migration
supabase link --project-ref <ref>
supabase db push

# 3. Deploy Edge Function
supabase functions deploy team-stats --project-ref <ref>
```

### Alternative: SQL Editor

Paste `sandbox.sql` into the [SQL Editor](https://supabase.com/dashboard/project/_/sql) and run. The dashboard `postgres` role has a 2-minute statement timeout â€” the script completes well under that. Then deploy the Edge Function via CLI or MCP.

## What's Inside

### Schema (8 tables + 1 view)

| Table | Purpose |
|-------|---------|
| `teams` | Top-level org unit, unique slug |
| `members` | Maps `auth.users` to teams with roles (owner/admin/member) |
| `projects` | Scoped to teams, various statuses |
| `tasks` | 18 seeded tasks with enums, JSONB metadata, due dates |
| `documents` | File metadata linked to Storage bucket |
| `comments` | Threaded via self-referential FK |
| `audit_log` | Auto-populated by trigger on tasks |
| `task_embeddings` | pgvector 384-dim embeddings for semantic search |
| `task_details` | View joining tasks/projects/teams/members (security_invoker) |

### RLS

Every table has RLS enabled with policies scoped to team membership. Patterns tested:

- `(select auth.uid())` cached subquery (not per-row)
- `TO authenticated` on all policies
- Security definer helpers in `private` schema
- Multi-hop: comments -> tasks -> projects -> teams -> members
- `auth.jwt()` claim extraction

### Seed Data

| Entity | Count |
|--------|-------|
| Auth users | 9 (in `auth.users` + `auth.identities`) |
| Teams | 3 (Engineering, Product, Design) |
| Members | 9 (3 per team, mixed roles) |
| Projects | 5 (active, archived, on_hold) |
| Tasks | 18 (all status/priority combos) |
| Documents | 4 |
| Comments | 8 (including threaded) |
| Embeddings | 18 (synthetic 384-dim vectors) |
| Audit log | Auto-generated by trigger |

All 9 users have passwords set to `password123` for testing.

### Additional Features

- **pgvector** with HNSW index and `match_tasks()` RPC
- **Storage** bucket `documents` with team-scoped RLS
- **Realtime** enabled on `tasks` and `comments`
- **Trigger** on tasks populating `audit_log`
- **Edge Function** `team-stats` calling `get_team_dashboard()` RPC

## RPC Functions

```sql
-- JSON dashboard for a team
select get_team_dashboard((select id from teams where slug = 'engineering'));

-- Text search
select * from search_tasks('auth', (select id from teams where slug = 'engineering'));

-- Vector similarity search
select * from match_tasks(
  (select embedding from task_embeddings limit 1),
  0.0,  -- threshold (use low value with synthetic vectors)
  5
);
```
